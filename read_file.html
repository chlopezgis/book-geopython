Lectura de datos Geoespaciales#

GeoPandas es una de las bibliotecas más utilizadas en Python para la manipulación y análisis de datos geoespaciales. Su función read_file permite cargar archivos vectoriales en distintos formatos y convertirlos en un GeoDataFrame, lo que facilita su procesamiento y análisis.

En este artículo, exploraremos su funcionamiento, los motores disponibles (Pyogrio y Fiona), y las estrategias de optimización para la lectura eficiente de datos.

Función “read_file”#

La función read_file permite leer archivos geoespaciales y cargarlos en un GeoDataFrame, que es la estructura de datos de GeoPandas para manejar información geográfica.

Sintaxis#

Su sintaxis es la siguiente:

Note

geopandas.read_file(filename, bbox=None, mask=None, columns=None, rows=None, engine=None, **kwargs )

Parámetros principales:

  • filename: (Obligatorio) Ruta del archivo a leer (local o URL).

  • layer: Nombre de la capa si el archivo tiene varias (ejemplo: en un GeoPackage).

  • bbox: Filtra los datos en la lectura usando un bounding box (minx, miny, maxx, maxy).

  • mask: Filtra los datos con una geometría de referencia (ejemplo: un polígono de corte).

  • rows: Número máximo de filas a leer (útil para grandes volúmenes de datos).

  • columns: Selección de columnas específicas para optimizar la carga.

  • engine: Define el motor de lectura. Desde GeoPandas 1.0.0, el valor predeterminado es pyogrio si está instalado; de lo contrario, se intenta con fiona

¿Cómo funciona read_file internamente?#

La función read_file utiliza Pyogrio como motor predeterminado para la lectura de archivos vectoriales desde GeoPandas 1.0.0. Este motor proporciona un rendimiento superior, especialmente al manejar grandes volúmenes de datos, y admite una amplia variedad de formatos gracias a su implementación optimizada de OGR.

En versiones anteriores a GeoPandas 1.0.0, el motor predeterminado es Fiona. Sin embargo, para mejorar el rendimiento en la lectura de archivos de gran tamaño, es posible utilizar Pyogrio especificando el parámetro engine="pyogrio".

Lectura de diferentes formatos vectoriales#

Antes de iniciar la lectura, importaremos GeoPandas junto con otras bibliotecas que utilizaremos a lo largo del tutorial.

import warnings

Lectura de una capa específica#

Algunos formatos vectoriales, como GeoPackage y ESRI GeoDatabase, pueden almacenar múltiples capas en un solo archivo. En estos casos, es necesario especificar la capa a leer; de lo contrario, GeoPandas cargará únicamente la primera capa por defecto.

En este ejemplo, leeremos una capa de robos almacenada en un GeoPackage. Para ello, primero listaremos las capas disponibles con gpd.list_layers() y así identificar el nombre exacto de la capa.

# Ruta del archivo GPKG de delitos
fileRobos = r'D:\Charlie\01_Cartografia\INEI\Seguridad\delitos.gpkg'

Lectura desde archivos ZIP#

También es posible leer archivos ZIP que contengan datos geoespaciales. Existen 2 casos para estos:

  1. Una sola capa: Ya sea comprimida directamente o dentro de una carpeta, su lectura es directa con solo indicar la ruta completa del archivo ZIP.

  2. Varias capas: Se debe especificar la ruta completa del archivo ZIP. Si las capas están dentro de una carpeta, también es necesario indicar su nombre.

Exploremos estos casos:

1. Una sola capa

Como se mencionó anteriormente, ya sea de forma individual o dentro de una carpeta, su lectura es directa, igual que la de cualquier archivo vectorial. A continuación, realizaremos la lectura de una capa única dentro de un archivo ZIP que contiene una carpeta.

Primero, exploraremos la estructura del archivo ZIP utilizando el comando tar desde la línea de comandos:

!tar -tf D:/Charlie/01_Cartografia/catastro.zip

data/
data/sectores.json

Se observa que el archivo comprimido contiene una carpeta llamada “data”, dentro de la cual hay un único archivo denominado “sectores.json”. Por lo tanto, podemos hacer la lectura directamente:

# Ruta del archivo ZIP de catastro
fileSectores = r'D:\Charlie\01_Cartografia\catastro.zip'

# Lectura de datos
sectores = gpd.read_file(fileSectores, encoding='UTF-8')

Lectura optimizada#

Cuando trabajamos con grandes volúmenes de datos, es fundamental aplicar técnicas que optimicen su lectura. Entre ellas, podemos filtrar la información para cargar solo los datos necesarios o utilizar pyogrio como motor de lectura, el cual ofrece un rendimiento superior en comparación con Fiona.

Filtrar por un BBOX#

El filtrado por BBOX es más eficiente en formatos que admiten indexación espacial y filtrado en la lectura, como GeoPackage y PostGIS. Estos formatos permiten aplicar el filtro directamente durante la consulta, evitando cargar en memoria geometrías fuera del área de interés.

En cambio, en formatos como Shapefile o GeoJSON, que no poseen indexación espacial avanzada, el filtrado por BBOX requiere cargar todas las geometrías antes de aplicar el filtro, lo que puede ralentizar el proceso y aumentar el uso de memoria.

En este ejemplo, aplicaremos el filtrado por BBOX a una capa de robos almacenada en un GeoPackage. Para ello, usaremos el GeoDataFrame de distritos, previamente cargado, para obtener el BBOX de un distrito de interés.

# Filtrar el distrito de Lince desde el 
# GeoDataFrame de "distritos"
lince = distritos[distritos.IDDIST=='150116']
lince.plot(figsize=(3,3));

Obtener el BBOX del distrito de Lince.

# Obtener el BBOX del distrito de Lince:
bboxLince = tuple(lince.total_bounds)
bboxLince

(-77.04858598599986,
 -12.091910229999883,
 -77.02281236099992,
 -12.078710979999869)

Leeremos la capa “robo_23” aplicando un filtro espacial basado en el BBOX del distrito de Lince.

# lectura de datos con filtro por bbox del distrito de Lince
robo_bbox_lince_23 = gpd.read_file(
                                fileRobos,        # filename
                                layer='robo_23', # layername
                                bbox=bboxLince    # cuadro delimitador
                    )

Visualizando los resultados

# Visualizacion
m = lince.explore(   
                tiles="CartoDB positron",
                style_kwds={'fillOpacity':0, 
                            'weight':4}
               )

robo_bbox_lince_23.explore(m=m,
                    marker_kwds={"color": "red", 
                                   "radius": 8}
                    )

Filtrar por máscara#

El filtrado por máscara (mask) es más eficiente en formatos que admiten indexación espacial y filtrado en la lectura, como GeoPackage y PostGIS. Estos formatos permiten aplicar el filtro directamente durante la lectura, evitando cargar datos innecesarios en memoria.

En contraste, formatos como Shapefile o GeoJSON, que no cuentan con indexación espacial avanzada, requieren cargar todos los datos en memoria antes de aplicar el filtro, lo que puede hacer que el proceso sea más lento y consuma más recursos.

En este ejemplo, aplicaremos el filtrado por máscara a una capa de robos almacenada en un GeoPackage. Para ello, utilizaremos el GeoDataFrame de lince, generado en el ejemplo anterior.

# lectura de datos con filtro por máscara del distrito de Lince
robo_lince_23 = gpd.read_file(
                                fileRobos,        # filename
                                layer='robo_23',  # layername
                                mask=lince        # cuadro delimitador
                            )

Visualizando:

# Visualizacion
m = lince.explore(   
                tiles="CartoDB positron",
                style_kwds={'fillOpacity':0, 
                            'weight':4}
               )

robo_lince_23.explore(m=m,
              marker_kwds={"color": "red", 
                           "radius": 8}
             )

Mejorar el rendimiento con Pyogrio#

Esta sección del tutorial aplica solo a versiones anteriores a GeoPandas 1.0.0, donde Fiona es la biblioteca predeterminada para la lectura de archivos vectoriales.

En estas versiones, es posible mejorar el rendimiento utilizando el parámetro engine="pyogrio", ya que, como se mencionó previamente, este motor proporciona una interfaz optimizada para la lectura y escritura de datos vectoriales mediante GDAL/OGR.

El rendimiento de pyogrio en comparación con Fiona depende de varios factores:

  • Formato del archivo: Pyogrio es más rápido formatos que admiten indexación espacial, como GeoPackage o Postgis. En Shapefile y GeoJSON, la ventaja es menor porque deben leerse completamente antes de aplicar filtros.

  • Tamaño del archivo: En archivos grandes (>1GB), Pyogrio puede ser hasta 5 veces más rápido que Fiona, mientras que en archivos pequeños la diferencia es mínima.

  • Uso de hilos: Pyogrio admite lectura paralela en sistemas multinúcleo, mejorando el rendimiento en el acceso a grandes volúmenes de datos.

Compararemos la lectura de datos del mismo archivo utilizando primero Fiona y luego Pyogrio.

# Ruta del archivo Geopackage:
file=r'D:\Charlie\01_Cartografia\catastro.gpkg'

# Crearemos una mascara para filtrar los datos del 
# distrito de San Juan de Lurigancho
sjl = distritos[distritos['IDDIST'] == '150132']

Lectura de datos con Fiona

# Tiempo de inicio
start = time.time()

# Lectura de datos utilizando fiona
data = gpd.read_file(file, 
                     layer='puertas', 
                     engine='fiona',
                     mask=sjl
                    )

# Imprimir tiempo de proceso y cantidad de registros
print("Tiempo con Fiona:", time.time() - start)
print('Registros:', data.shape)

Tiempo con Fiona: 27.717732667922974
Registros: (226230, 16)

Lectura de datos con Pyogrio

# Tiempo de inicio
start = time.time()

# Lectura de datos utilizando pyogrio (por defecto)
data = gpd.read_file(file, 
                     layer='puertas',
                     mask=sjl
                    )

# Imprimir tiempo de proceso y cantidad de registros
print("Tiempo con Pyogrio:", time.time() - start)
print('Registros:', data.shape)

Tiempo con Pyogrio: 9.69790267944336
Registros: (226230, 16)

Conclusión#

La lectura de datos geoespaciales es un paso fundamental en cualquier flujo de trabajo GIS en Python, y GeoPandas simplifica este proceso con la función read_file, que permite cargar archivos vectoriales de manera sencilla. Sin embargo, cuando se trabaja con grandes volúmenes de datos, es esencial optimizar la lectura para mejorar el rendimiento y el uso de memoria.

El uso de formatos con indexación espacial, como GeoPackage o PostGIS, junto con técnicas como el filtrado por BBOX o máscara, ayuda a limitar la cantidad de datos cargados en memoria. Además, en versiones posteriores a GeoPandas 1.0.0, el motor de lectura influye directamente en la eficiencia: Fiona ofrece estabilidad y compatibilidad, mientras que Pyogrio permite una lectura más rápida de grandes volúmenes de datos.

La elección del enfoque adecuado dependerá del tamaño de los datos, el formato del archivo y los requisitos de procesamiento, lo que permitirá optimizar significativamente el rendimiento en la lectura de datos geoespaciales.
